# RA-SOURCER: 소재 발굴 + 지문 초안 생성기

**OMC 전제.** 이 스킬은 Claude Code의 Agent 도구로 opus 서브에이전트를 spawn한다. OMC 미설치 시 자연어 트리거가 동작하지 않는다.

LEET 추리논증의 **병목 단계**. 영역 지정 또는 키워드를 받아 출제 가능한 지문 초안을 생성한다.
소재가 부실하면 이후 analyze/design/review가 전부 무너진다. 여기에 시간을 투자한다.

## When to Activate

- 사용자가 소재만 주고 지문부터 문항까지 만들어달라고 할 때
- 영역(법/인문/사회/과학/논쟁) 지정 요청
- 세트 단위 요청의 각 라운드 시작 시

## Workflow

### Step 1: 프롬프트 로드

- `leet_ra/prompts/sourcer.md` 전문을 Read한다.

### Step 1b: 영역별 기출 지문 RAG 주입 (필수)

사용자 입력에서 영역을 추출한 뒤, 해당 영역의 기출 지문 2~3개를 few-shot으로 주입한다.

**영역 → leet_type 매핑**:

| 영역 | leet_type 매핑 |
|------|---------------|
| 법 (규범) | D1, D2, D3 |
| 인문 | A1, A2, A3 |
| 사회 | C1, C2, E2 |
| 과학 | C1, C2, C3 |
| 논쟁 | E1, E2 |

**주입 절차**:

1. Read `leet_ra/data/2025_merged.json` → `questions` 배열 (가중치 90% — 우선 탐색)
2. Read `leet_ra/data/2024_merged.json` → `questions` 배열 (가중치 10% — 보충)
3. `leet_type`이 매핑 범위에 속하고 `passage_text` 길이가 400~1200자인 questions 중 2~3개 선별 (2025 우선, 부족 시 2024 보충)
4. 각 선별된 질문에서 다음 필드를 프롬프트에 포함:
   - `leet_type`, `passage_topic`, `passage_components`, `question_type`
   - `passage_text` 전문
5. **세트 모드**에서는 이전 라운드의 `passage_topic` 리스트를 수집하여 중복 방지용으로 프롬프트에 포함

### Step 2: 에이전트 Spawn

```
Agent(
  name="ra-sourcer",
  description="LEET 추리논증 소재 발굴 + 지문 생성",
  model="opus",
  mode="auto",
  prompt="{sourcer.md 전문}\n\n"
        "## 입력\n\n{영역 지정 또는 소재 키워드}\n\n"
        "## Few-shot 참조 — 최근 기출 동일 영역 지문 2~3개\n\n"
        "{Step 1b에서 선별된 passage_text 블록들}\n\n"
        "## 중복 방지 (세트 모드 시)\n\n"
        "{이전 passage_topic 리스트 or '없음'}"
)
```

### Step 3: 결과 사용자 승인

지문 초안을 사용자에게 제시하고 **승인을 받은 후** ra-analyze로 넘긴다.

## 지문 빌드 3대 원칙 (절대 규칙)

1. **일반화**: 고교 수준 배경지식으로 지문 자체는 편하게 이해 가능. 변별은 선지 판단에서 발생.
2. **실명 금지**: 마르크스·칸트·하버마스 등 실제 학자명 절대 금지. '갑', '을', '병', `<이론>`, 'X국' 등으로 일반화.
3. **한국어 전용**: 영어·외국어·학술 약어 금지.

## 영역별 절대 금지

| 영역 | 절대 금지 / 권장 소재 |
|------|--------------------|
| **법 (규범)** | 민법·형법·행정법 등 변호사시험 빈출 소재 ❌. **특별법 영역(환경/교육/소비자/전자상거래 등) 사용.** |
| **인문** | 기호 논리학(∀, ∃, →) 과도 사용 ❌. '대우' 정도만 허용. 전공 개념은 지문 내 충분한 설명 필수. |
| **사회/과학** | 특별한 금지 없음. 소재 다양화에 초점. |

## 품질 체크

- 실명/이론명 **0개**
- 한국어 외 문자 **0개**
- 출제 가능성 ≥ 3개 포인트
- 영역에 맞는 빌드 구성요소 사용 (P/V/R/C/PR/AS/MR/MK/T)
- 고교 배경지식으로 지문 이해 가능

## Notes

- sourcer는 **'환각' 허용**. 논리가 풍부하고 말만 되면 실제 법령/논쟁이 아니어도 된다.
- 규정(R) 작성 시: 시간 표현 명확화, '아니한다' → '않는다', '만 원' 띄어쓰기 (sourcer.md 참조).
- **기출 가중치 90/10.** 2025 우선, 2024는 보충.
