# RA-DESIGN: 선지 설계기 (Agent B)

Agent A(analyze)가 추출한 출제 포인트·W코드·논증 구조를 바탕으로 실전 LEET 수준의 **발문 + 선지 + 해설**을 설계한다.

## When to Activate

- analyze 실행 후, 문항 설계를 요청할 때
- "문항 만들어줘", "선지 설계해줘" 등
- 풀 파이프라인의 3단계

## Workflow

1. `leet_ra/prompts/design.md` 파일을 Read한다.
2. 이전 대화의 analyze 출력을 추출한다.
3. **opus 서브에이전트를 spawn**한다:

```
Agent(
  name="ra-design",
  description="LEET 추리논증 선지 설계",
  model="opus",
  mode="auto",
  prompt="{design.md 전문}\n\n## Agent A 분석 데이터\n\n{analyze 결과}\n\n## 원본 지문\n\n{지문}"
)
```

4. 결과(발문 + 선지 + 해설)를 사용자에게 전달한다.

## 선지 구성 핵심 규칙

- **Rule 4 (절대)**: 매력도 하 선지 출력 금지. 지문을 한 번 읽으면 즉시 배제 가능한 오답은 폐기.
- **Rule 16**: 지문 표현 2~3어절 이상 동일 인용 금지. 반드시 패러프레이징. (규범 영역 예외 가능)
- **W코드 다변화**: 한 문항 내에서 같은 W코드 반복 금지. 서로 다른 W코드 2~3종 배치.
- **정답 유일성 자가 검증**: 설계 후 모든 선지를 역으로 풀어 정답이 유일한지 확인.
- **지문 근거 제한**: 정답/오답 판정 근거는 반드시 지문 내. 배경지식으로만 풀리는 선지 금지.
- **발문 고정**: 기출의 전형적 발문을 그대로 사용. 변형 금지.

## 품질 체크

- 출제 포인트 **전량 소모** (미소모 = 설계 결함)
- 매력도 하 선지 **0개** (품질 게이트)
- 오답 W코드 **2~3종 이상** 다변화
- 역풀이 정답 유일성 통과
- 모든 선지 근거가 지문 내
- 해설 논리 완결성 (순환 논증·비약 없음)

## 기출 발문 패턴

| 영역 | 전형 발문 |
|------|----------|
| 법 적용 (D1) | "~에 따라 바르게 추론한 것은?" |
| 논리 퍼즐 (B3) | "옳지 않은 것은?" |
| 견해 비교 (D3/E2) | "견해에 대한 평가로 옳은 것은?" |
| 논쟁 평가 (E1) | "진술로 옳지 않은 것은?" |

## Notes

- design 후에는 반드시 **ra-review를 독립 세션에서 spawn**한다.
- 보기 조합형(ㄱㄴㄷ) 문항은 각 진술의 옳고 그름이 지문에서 일의적으로 확정되어야 한다.
- AP코드(매력도 패턴) 11종과 선지 구성 규칙 16개는 `leet_ra/data/choice_logic_analysis.json` 참조.
